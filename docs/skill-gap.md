# 역량 격차 분석

## 1. 핵심 취약점

| 영역 | 현재 상태 | 목표 상태 |
|---|---|---|
| Cloud 환경 구조 | 생소함 — 실무에서 온프레미스 중심, Cloud 아키텍처 경험 없음 | 주요 클라우드의 컴퓨트/네트워크/스토리지/IAM 구조를 스스로 설계·설명 가능 |
| AI를 활용한 개발 | 초급 — AI 앱 개발·배포 경험은 있으나 깊이 부족 | AI 기반 개발 도구·워크플로우를 실무 수준으로 활용, AI 인프라 설계 이해 |

## 2. 보유 역량 (참고: [career-background.md](../reference/career-background.md))

- Linux/Unix 기반 하드웨어·시스템 이해
- 네트워크 구조 파악
- VTL/PTL·서버·스토리지 구조 실무 경험
- U2L(Unix to Linux) 전환 트러블슈팅 (초급)
- AI 앱 개발·배포 (초급)
