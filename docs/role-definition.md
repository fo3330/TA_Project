# Technical Architect (AI + Cloud) 직무 정의 (초안)

> 이 문서는 초안이며, 실제 채용공고·현직자 인터뷰 등을 통해 계속 보강할 것.

## 1. 직무 개요

Technical Architect(TA)는 비즈니스 요구사항을 기술 구조로 변환하고, 시스템 전체의 설계 방향(아키텍처)을 결정·검증·문서화하는 역할. 이 저장소가 목표로 하는 TA는 특히 **AI 워크로드와 Cloud 인프라를 결합한 설계**에 강점을 둔 포지션을 의미.

## 2. 일반적으로 요구되는 역량 (가설 — 검증 필요)

- **Cloud 아키텍처**: IaaS/PaaS/SaaS 구조 이해, 주요 클라우드(AWS/GCP/Azure) 핵심 서비스(컴퓨트, 네트워크, 스토리지, IAM), 멀티/하이브리드 클라우드 설계, 비용·가용성·보안 트레이드오프 판단
- **AI/ML 인프라**: LLM/ML 모델 서빙 구조, GPU 리소스 관리, 벡터DB·RAG 파이프라인, AI 에이전트 아키텍처, MLOps 기초
- **시스템 설계 일반**: 확장성/가용성/장애복구 설계, IaC(Terraform 등), CI/CD, 옵저버빌리티
- **소프트 스킬**: 이해관계자와의 커뮤니케이션, 설계 문서화(ADR 등), 의사결정 근거 제시

## 3. 이 저장소 사용자의 강점으로 연결되는 지점

- 시스템 운영 3년 + 인프라 구축 사업 2년 → 온프레미스 인프라·네트워크·스토리지 구조에 대한 실무 감각은 Cloud 아키텍처 학습의 기반이 될 수 있음
- U2L(Unix to Linux) 전환 트러블슈팅 경험 → 이기종 시스템 마이그레이션 설계 역량과 연결 가능
- 경력 상세: [../reference/career-background.md](../reference/career-background.md)

## 4. 검증이 필요한 질문 (추후 리서치)

- [ ] 실제 채용공고 5~10건 수집·비교 (요구 스킬, 자격증, 경력 연차)
- [ ] AWS/GCP/Azure 중 우선 학습할 벤더 선택 근거 마련 (현재 실무 환경, 채용 시장 수요 기준)
- [ ] "AI Solutions Architect"류 직무와 전통적 TA의 차이 정리
- [ ] 목표 회사/업계군 후보 리스트업 (스타트업 vs 대기업 vs 클라우드 벤더)
