# 현재 프로젝트의 AI 활용 기록

> 2026-07 작성. 상세 경력 타임라인은 [reference/career-background.md](../reference/career-background.md) 참고.

## 1. 프로젝트 개요

- **프로젝트명**: 국립대학자원관리시스템(KORUS) HCI 구축 전환 사업
- **목적/배경**: 전국 국립대학 DB 42식을 Oracle 12c에서 19c로 업그레이드하고, Jeus7·Damo·ecross 등 전환 솔루션 21식을 구축·이관. 12c→19c 마이그레이션 과정에서 기존 소스 코드·DB 프로시저의 SQL 구문이 19c 문법과 맞지 않아 발생한 Syntax error를 해결하는 것이 핵심 과제.
- **나의 역할**: HCI 구축 전환팀 소속, AA(Application Architect) 투입 이슈 해결 과정의 보조 (2026.01 ~ 2026.07, 6개월)

## 2. AI 활용 내역

| 활용 영역 | 사용한 AI/도구 | 구체적 활용 방식 | 성과/배운 점 |
|---|---|---|---|
| DB 마이그레이션 이슈 해결 | Claude | 12c→19c 전환 시 발생한 SQL Syntax error의 원인을 분석하고, 소스 코드 및 DB 프로시저 고도화 작업을 보조받음 | 트러블슈팅 소요 시간 단축, AI를 활용한 개발의 첫 실전 경험 |
| 애플리케이션 개발 보조 | Claude | AA(Application Architect)가 투입된 이슈 해결 과정에서 보조 개발 수행 | KORUS 실제 시스템의 애플리케이션 구조와 이슈 해결 방식을 파악 |
| 개인 프로젝트·포트폴리오 작성 | Claude Pro (구독) | 이직 준비용 학습 로드맵 저장소(이 TA_project) 및 포트폴리오 작성에 활용 | 이 저장소 자체가 결과물 |

## 3. TA 직무 관점에서의 연결점

- 온프레미스 DB 마이그레이션 트러블슈팅 경험을 클라우드 마이그레이션(예: RDS로의 이관) 시나리오로 확장하면 [Phase 3](../roadmap/phase-3-portfolio-practice.md) 포트폴리오 프로젝트로 재구성할 수 있음
- AA 보조 경험은 아키텍처 설계 문서화(ADR) 연습의 출발점이 될 수 있음
- 부족한 점: 클라우드 환경에서의 실제 마이그레이션 경험은 아직 없음 → Phase 3에서 직접 실습 필요. 상세 격차 분석은 [reference/career-background.md](../reference/career-background.md)의 "직무 정의 대비 준비 과제" 참고
