# 자기소개서 초안 (Technical Architect / AI+Cloud 지원용)

> 대시보드 앱의 "자기소개서" 탭 5개 항목과 동일한 내용. [reference/career-background.md](reference/career-background.md)의 상세 경력과 [docs/role-definition.md](docs/role-definition.md)의 요구 역량을 근거로 작성. 목표 회사가 정해지면 해당 채용공고에 맞춰 다시 다듬을 것.

## 1. 지원동기

저는 2022년부터 국가정보자원관리원 산하 여러 사업에서 대민 서비스 시스템의 운영과 인프라 구축을 담당해왔습니다. 서버 180여 식, 스토리지 10여 식 규모의 실운영 환경을 다루며 안정성의 무게를 몸으로 배웠고, 네트워크 장비 교체와 백업(VTL/PTL) 인프라 구축을 거치며 온프레미스 인프라의 운영과 구축 전 영역을 폭넓게 경험했습니다. 그리고 올해 국립대학자원관리시스템(KORUS) 프로젝트에서 처음으로 AI(Claude)를 실무에 활용해 DB 마이그레이션 이슈를 해결하면서, 인프라를 다루는 방식 자체가 AI로 인해 바뀌고 있다는 것을 체감했습니다. 온프레미스에서 쌓아온 인프라 감각을 클라우드 환경으로 확장하고, 여기에 AI를 설계 도구이자 설계 대상으로 함께 다루는 Technical Architect가 되고자 이 방향을 선택했습니다.

## 2. 강점 및 차별화 포인트

가장 큰 강점은 "실제로 운영해본 사람의 감각"입니다. 서버·스토리지·네트워크·백업 장비를 이론이 아니라 장애 대응과 현장 실사를 통해 배웠기 때문에, 클라우드 개념을 학습할 때도 실제 트래픽과 장애 상황을 기준으로 이해할 수 있습니다.

예를 들어 소산 VTL을 백업 VTL로 용도 변경하고 PTL을 신규 구축하면서 백업 솔루션의 구조를 완전히 이해하게 되었는데, 이는 클라우드의 가용성·재해복구(HA/DR) 아키텍처를 학습할 때 그대로 적용할 수 있는 감각입니다. 또한 노후 L3 장비를 Cisco Nexus로 교체하며 네트워크 허용/차단 신청과 실사를 직접 진행한 경험은 클라우드 보안그룹·IAM 개념을 빠르게 체화하는 기반이 되고 있습니다.

## 3. 경력 하이라이트 (STAR)

**국립대학자원관리시스템(KORUS) DB 마이그레이션 트러블슈팅**

- **상황(S)**: KORUS HCI 구축 전환 사업에서 전국 국립대학 DB 42식을 Oracle 12c에서 19c로 업그레이드하는 작업 중, 기존 소스 코드와 DB 프로시저의 SQL 구문이 19c 문법과 맞지 않아 다수의 Syntax error가 발생했습니다.
- **과제(T)**: 유지관리 계약이 되어 있지 않은 솔루션까지 포함해, 짧은 기간 안에 원인을 파악하고 소스·프로시저를 고도화해야 했습니다.
- **행동(A)**: 별도의 테스트 서버를 구성해 이슈를 먼저 재현하고, AA(Application Architect)가 투입된 해결 과정에 보조로 참여하며 AI(Claude)를 활용해 원인 분석과 코드 수정 방향을 함께 검토했습니다.
- **결과(R)**: 트러블슈팅 소요 시간을 단축했고, 이 경험을 계기로 Claude Pro를 구독해 개인 프로젝트와 커리어 전환 준비에도 AI를 본격적으로 활용하기 시작했습니다.

**백업 인프라(VTL/PTL) 구축 전환**

업무망 소산 VTL을 백업 VTL로 용도 변경하고, 백업센터(공주)에 소산 VTL을 신규 구축·클라우드 소산 VTL을 600TB 증설하는 작업을 맡아 기간 내 과업을 완료했고, 이를 통해 고객 신뢰도를 높인 경험도 있습니다.

## 4. 극복 경험

가장 어려웠던 순간은 KORUS 프로젝트의 DB 마이그레이션 과정에서 예상보다 훨씬 많은 Syntax error가 쏟아졌을 때였습니다. 그동안은 인프라 운영·구축 위주로 일해왔기 때문에 애플리케이션·DB 프로시저 수준의 문제 해결은 익숙하지 않았습니다.

테스트 서버를 별도로 구성해 이슈를 하나씩 재현하고, AA의 해결 과정을 보조하며 AI(Claude)를 함께 활용해 원인을 좁혀나가는 방식으로 접근했습니다. 그 결과 문제를 해결했을 뿐 아니라, 인프라 담당자로서의 시야를 애플리케이션·AI 활용까지 넓히는 계기가 되었습니다.

## 5. 입사 후 포부

입사 후에는 지금까지 쌓아온 온프레미스 인프라 운영·구축 경험을 기반으로 Cloud 아키텍처(AWS/GCP/Azure)와 AI 인프라(RAG, 모델 서빙 등)를 체계적으로 학습해, 온프레미스와 클라우드, AI 워크로드를 모두 이해하고 설계할 수 있는 Technical Architect로 성장하고 싶습니다. 특히 실제 운영 경험을 바탕으로, 이론에 그치지 않고 장애와 트레이드오프를 고려한 현실적인 아키텍처를 제안하는 것을 목표로 합니다.
