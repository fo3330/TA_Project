# Phase 1 — Cloud 환경 구조 기초 다지기

## 목표

- Cloud 환경 자체에 대한 생소함 해소 (최대 취약점)
- 최소 1개 클라우드 벤더의 핵심 서비스를 스스로 설명할 수 있는 수준 도달

## 체크리스트

- [ ] 학습 우선 벤더 선정 (AWS / GCP / Azure 중 1개, 근거는 [docs/role-definition.md](../docs/role-definition.md)에 기록)
- [ ] IaaS/PaaS/SaaS 개념 및 클라우드 공동 책임 모델 학습
- [ ] 컴퓨트(VM/컨테이너/서버리스) 핵심 서비스 학습
- [ ] 네트워크(VPC, 서브넷, 로드밸런서, DNS) 핵심 개념 학습 — 기존 네트워크 실무 경험과 연결지어 정리
- [ ] 스토리지(오브젝트/블록/파일) 및 데이터베이스 서비스 개요 학습
- [ ] IAM/보안 기초 (권한, 정책, 네트워크 보안그룹)
- [ ] IaC 개념 및 Terraform 등 도구 하나 맛보기 실습
- [ ] 선택한 벤더의 입문 자격증(예: AWS Cloud Practitioner) 취득 여부 결정 및 진행
