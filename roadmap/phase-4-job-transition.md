# Phase 4 — 이직 준비 및 실행

## 목표

- 쌓은 역량과 포트폴리오를 바탕으로 실제 이직 프로세스 진행

## 체크리스트

- [ ] 목표 회사/업계군 리스트업 ([docs/role-definition.md](../docs/role-definition.md) 검증 결과 반영)
  - 방법: 원티드/사람인/링크드인에서 "Technical Architect" "Solutions Architect" "Cloud Architect" 채용공고 검색·스크랩
- [ ] 이력서를 TA(AI+Cloud) 직무 기준으로 재작성
  - 방법: 채용공고에 반복 등장하는 키워드를 반영, 성과는 정량 지표(예: 장애 대응 시간 단축 %)로 표현
- [ ] 포트폴리오/GitHub 링크 정리 및 이력서에 반영
  - 참고: [Phase 3](phase-3-portfolio-practice.md) 산출물
- [ ] 실제 채용공고 기준 역량 재점검 ([docs/skill-gap.md](../docs/skill-gap.md) 갱신)
- [ ] 네트워킹 (현직자 커피챗, 커뮤니티 참여 등)
  - 방법: AWS User Group Korea, 링크드인 등에서 현직 TA/아키텍트에게 커피챗 요청
- [ ] 지원 시작
- [ ] 면접 대비 (아키텍처 설계 문제풀이, 경력 스토리 정리)
  - 책: 『System Design Interview』(Alex Xu)
  - 방법: 경력 스토리는 STAR 기법(Situation-Task-Action-Result)으로 정리
