### 국립대학자원관리시스템(KORUS) HCI 구축 전환 사업 — 2026.01~2026.07 (6개월)

- **직무/역할**: HCI 구축 전환팀
- **주요 업무**: 전국 국립대학 DB 42식을 Oracle 12c에서 19c로 업그레이드하고, Jeus7·Damo·ecross 등 전환 솔루션 21식을 구축·이관. 12c→19c 마이그레이션 과정에서 기존 소스 코드와 DB 프로시저에 사용된 SQL 구문이 19c 문법과 맞지 않아 Syntax error가 다수 발생했고, 이를 해결하기 위해 소스 및 DB 프로시저 고도화 작업을 직접 수행.
- **성과·배운 점**: 테스트 서버를 별도로 구성해 이슈를 먼저 재현·파악하는 방식으로 실전 트러블슈팅 경험을 쌓음. 테스트 서버에서 솔루션을 직접 기동·테스트하며 문제 해결 능력을 키움. AA(Application Architect)가 투입된 이슈 해결 과정에 보조로 참여하며 AI(Claude)를 활용한 애플리케이션 개발을 처음 경험함. KORUS 실제 시스템의 애플리케이션 구조와 이슈 해결 방식을 파악. 유지관리 계약이 체결되지 않은 솔루션의 이관·기동까지 직접 처리하며 역량이 향상됨. 이 경험을 계기로 Claude Pro를 구독해 개인 프로젝트와 포트폴리오 작성에 AI를 활용하기 시작함.

### 국가정보자원관리원 자원통합 구축 사업 1-3차 — 2025.06~2026.01 (8개월)

- **직무/역할**: 백업팀
- **주요 업무**: 업무망 소산 VTL(FALCON)을 백업 VTL(DD9910)로 용도 변경. 백업센터(공주)에 소산 VTL(DD9910)을 신규 구축하고 클라우드 소산 VTL을 600TB 증설. 소산 PTL(IBM TS4500) 구축. 기존 통합백업서버(Oracle·AIX 기반, 마스터서버 6식 + 패러렐 서버 8식)를 신규 통합백업서버(Linux 기반)로 전환.
- **성과·배운 점**: 소산 VTL을 백업 VTL로 용도 변경하며 VTL·서버·백업 솔루션 전반에 대한 이해도를 습득. VTL 신규 구축과 증설을 통해 백업 인프라 구조에 대한 이해를 심화. PTL(IBM TS4500) 구축 경험으로 PTL 장비에 대한 이해를 확보. 백업 장비(VTL/PTL)에 연결된 스토리지의 WWN을 조사하고 실제 장비를 실사하며 스토리지 점검·조사 능력을 습득. 이전 2-2차 사업의 네트워크 경험을 살려 전 장비에 대한 네트워크 조사·실사 및 허용/차단 신청을 진행하며 네트워크 이해도를 한층 높임. 기간 내 과업을 완료해 고객 신뢰도를 향상시킴.

### 국가정보자원관리원 자원통합 구축 사업 2-2차 — 2024.11~2025.03 (5개월)

- **직무/역할**: 레거시 전환(네트워크)
- **주요 업무**: 기상청·농림부·법무부의 노후 L3 장비(8식)를 Cisco Nexus 9504에서 Nexus 93180YC로 교체하는 구축 작업 수행.
- **성과·배운 점**: 네트워크 기본 지식을 습득하고 상단(코어)·하단(액세스) 구조에 대한 이해도를 높임. 구축을 위한 현장 실사를 진행하며 꼼꼼함과 정확성을 습득. 네트워크 컨피그 확인 및 변경 기술을 익힘. 2차 사업에 투입되어 짧은 기간 내 과업을 완료하며 고객 신뢰도를 향상시킴.

### 국가정보자원관리원 정보시스템 2군 — 2022.01~2024.05 (2년 5개월)

- **직무/역할**: 국세운영팀 대민UNIX 부서
- **주요 업무**: 서버 약 180식, 스토리지 약 10식 규모의 인프라를 대상으로 국세청 대민 서비스(세정 신고, 현금영수증, 근로장려세제, 대민 포털, 모바일 포털, 연말정산, e-세로, 대민 관리) 및 개발·검증(대민), DR(재해복구) 서비스 운영을 담당.
- **성과·배운 점**: AIX와 Linux를 중급 수준으로 능숙하게 다룰 수 있게 됨. 하드웨어 장애 발생 시 서버와 콘솔을 통해 장애 지점을 확인하는 능력을 습득. 국가정보자원관리원이라는 대규모 인프라 전산실을 운영해본 경험을 쌓음. 실제 운영 중인 서버를 다루며 안정성과 꼼꼼함을 몸에 익힘.

### 전체 흐름 메모

2022년 국세운영팀 대민 UNIX 시스템 운영을 시작으로, 네트워크 장비(L3) 구축 전환, 백업 인프라(VTL/PTL) 구축 전환까지 온프레미스 인프라의 운영·구축 전 영역을 순차적으로 경험. 2026년부터는 국립대학자원관리시스템(KORUS) 프로젝트에서 DB 마이그레이션 트러블슈팅과 AI(Claude)를 활용한 애플리케이션 개발을 함께 경험하며, 인프라 운영자에서 AI를 개발 도구로 활용하는 방향으로 역할이 확장되는 계기가 됨. 이 흐름은 "온프레미스 인프라 실무 경험 + AI 활용 개발"을 결합한 TA(Technical Architect) 방향과 자연스럽게 이어짐.

## 직무 정의 대비 준비 과제 (2026-07 작성)

> [docs/role-definition.md](../docs/role-definition.md)의 "요구 기술 스택" 절과 위 경력을 대조한 분석.

### 이미 강점으로 연결되는 부분

- 서버·스토리지·네트워크·백업 인프라의 운영과 구축 전 영역을 실무로 경험 → [Phase 1](../roadmap/phase-1-cloud-foundations.md) 학습을 훨씬 빠르게 흡수할 기반
- 장애 대응, 현장 실사, 트러블슈팅 등 안정성·꼼꼼함이 몸에 배어 있음
- 네트워크 허용/차단 신청, 실사 경험 → Cloud IAM·보안그룹 개념 학습 시 빠르게 체화 가능
- 백업/DR 서비스 운영 경험 → Cloud의 가용성·재해복구(HA/DR) 아키텍처 설계 학습과 직접 연결
- KORUS 프로젝트에서 AI(Claude)를 활용한 개발을 이미 시작함 → [Phase 2](../roadmap/phase-2-ai-driven-dev.md)의 출발점을 이미 확보

### 더 준비해야 할 것

1. **Cloud 실무 경험 전무** — AWS/GCP/Azure를 실제로 사용해본 이력이 없음 → Phase 1을 최우선으로, 온프레미스 지식을 클라우드 개념에 매핑하며 학습
2. **IaC(Terraform) 실무 경험 없음** → Phase 1 체크리스트의 Terraform 실습 필요
3. **컨테이너(Docker/Kubernetes) 경험 없음** (전통적 서버/VM 중심 운영) → Phase 1~2에서 병행 학습
4. **프로그래밍/스크립팅 언어(Python 등) 활용 경험이 드러나지 않음** (DB 프로시저 SQL 수준) → Phase 2 학습과 함께 자동화 스크립트 작성 연습 필요
5. **RAG·LLM 아키텍처, 모델 서빙 등 AI 인프라 지식은 이제 막 시작 단계** (Claude 활용 애플리케이션 개발 1건) → Phase 2 심화 필요
6. **아키텍처 설계 문서화(ADR 등) 작성 경험 없음** (주로 운영·구축 실무 중심) → Phase 3에서 처음 연습
7. **클라우드 관련 자격증 없음** → Phase 1 학습 후 AWS Cloud Practitioner부터 취득 권장

### 포트폴리오화 아이디어

- KORUS 프로젝트의 **Oracle 12c→19c DB 마이그레이션 트러블슈팅 경험**을 클라우드 마이그레이션(예: RDS로의 이관) 시나리오로 확장해 [Phase 3](../roadmap/phase-3-portfolio-practice.md) 포트폴리오 프로젝트로 재구성
- AA(Application Architect) 보조 경험을 아키텍처 설계 문서화(ADR) 연습의 출발점으로 활용
